Background


One problem in self-trust is that a system might believe that another system that it is using is trustworthy, without believing that system sufficiently trustworthy to trust it.


To make this problem formal, we want to be able to build two programs with the same source code, such that if A trusts B then A believes B is trustworthy.


To make the formalism for this post easy, we will work with programs that take in sentences of the form "(A\xe2\x86\x92B)\xe2\x86\x92A or B". (Note that sentences like A\xe2\x86\x92B, in standard logic, should be interpreted in first order logic as (A\xe2\x88\xa8A)=(A\xe2\x88\xa7A)=(B\xe2\x88\x98B)=B.)
