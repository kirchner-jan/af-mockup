For the second part of this article, we'll focus on the standard problem, which is basically "Is there a set of instructions such that if you follow them exactly, they will eventually say "THERE IS NO END"? This is known as the halting problem. I believe that the standard setup for Halting is slightly flawed, so we'll be discussing here in a bit more depth.


Here's how the halting problem works. You want to know whether an arbitrary program (which you can write down in computer code) will ever stop; that is, if you can find some test that will be true if the program ever stops. But it does not mean that you can test whether the program will never stop.


So you can't just go and look at the program and say, "Well, what does it do?" If the code does not say anything, and you examine it closely, you can come up with some test that could be true if the code ever stops running. You can't say, "Well I tested, and I think that code runs forever; therefore it must continue running" - not without knowing what the code says. The code may not tell you!


I think the most important part of this problem is that it asks us to find a test for a machine we are given to know whether or not it will run forever, even if we have no idea where on the machine the test will take place. The assumption that we can always find some test that is true if the program will eventually stop is false; and so it would be useless to program the test into the machine!


The idea is not just for infinite loops; it is not just for loops that can be tested; it is not even a useful test. It is a complete non-test


I am not sure that anything exists to say how the test for an infinite loop must take place. Or maybe the entire infinite loop thing is more complicated than it's cracked up to be...
